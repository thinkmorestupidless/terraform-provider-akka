# Implementation Plan: Akka Platform Terraform Provider

**Branch**: `001-akka-terraform-provider` | **Date**: 2026-04-30 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/001-akka-terraform-provider/spec.md`

## Summary

Build a Terraform provider for the Akka platform that enables infrastructure engineers to declaratively manage Akka projects, services, secrets, routes, hostnames, and role bindings. The provider wraps the `akka` CLI binary installed on the host, using its JSON output mode (`-o json`) to drive all CRUD operations. Authentication uses the documented `AKKA_TOKEN` environment variable. All resources follow the `terraform-plugin-framework` v1.19.0 patterns for lifecycle management, drift detection, sensitive field handling, and import support.

**Structure Decision**: Single Go module following the standard `terraform-provider-scaffolding-framework` layout. Provider logic split between `internal/provider/` (Terraform schema + lifecycle) and `internal/akka/` (CLI wrapper client). This separation keeps Terraform framework concerns isolated from CLI invocation concerns, enabling independent unit testing of both layers.

## Implementation Phases

### Phase A: Foundation (CLI Client + Provider Shell)

**Goal**: A provider that can authenticate and connect to Akka; no resources yet.

1. Initialize Go module with `go mod init github.com/thinkmorestupidless/terraform-provider-akka`
2. Add dependencies: `terraform-plugin-framework`, `terraform-plugin-log`, `terraform-plugin-go`
3. Implement `internal/akka/client.go`:
   - `AkkaClient` struct: `BinaryPath`, `Token`, `Organization`, `DefaultProject`
   - `NewClient(binaryPath, token, org, defaultProject string) *AkkaClient`
   - `Run(ctx context.Context, args ...string) ([]byte, error)` — wraps `exec.CommandContext`, sets `AKKA_TOKEN` and `AKKA_DISABLE_PROMPTS=true` in env, appends `--organization <org>` to all commands, captures stdout/stderr
   - Error handling: detect not-found responses vs. other API errors; define `NotFoundError` type
4. Implement `internal/akka/errors.go`: `NotFoundError`, `IsNotFound(err) bool`
5. Implement `internal/provider/provider.go`:
   - Provider schema: `organization` (required), `token` (optional, sensitive), `project` (optional)
   - `Configure()`: `exec.LookPath("akka")` → error if missing; token from attr or `AKKA_TOKEN`; construct `AkkaClient`; store in `resp.ResourceData` and `resp.DataSourceData`
   - Empty `Resources()` and `DataSources()` lists (filled in later phases)
6. Implement `main.go` entry point
7. Write unit tests for `AkkaClient.Run()` using a mock binary (a shell script that echoes JSON)
8. Build and verify: `go build ./...`

**Deliverable**: `terraform init` succeeds (with dev override); provider configure errors correctly when `akka` is missing or token is absent.

### Phase B: Project Resource (P1 story 1)

**Goal**: Full CRUD for `akka_project` with drift detection and import.

1. Implement `internal/akka/project.go`:
   - `ProjectModel` struct (JSON fields from `akka projects get -o json`)
   - `CreateProject(ctx, name, description, region string) (*ProjectModel, error)`
   - `GetProject(ctx, name string) (*ProjectModel, error)`
   - `UpdateProject(ctx, name, description string) (*ProjectModel, error)`
   - `DeleteProject(ctx, name string) error`
   - `ListProjects(ctx) ([]ProjectModel, error)`
2. Implement `internal/provider/project_resource.go`:
   - Schema: `id` (computed), `name` (required, forces recreate), `description`, `region`, `organization`
   - `Create`: call `CreateProject`, set state
   - `Read`: call `GetProject`; on `NotFoundError` → `resp.State.RemoveResource(ctx)`
   - `Update`: call `UpdateProject` (description only; name change triggers recreate via `PlanModifiers: []planmodifier.String{stringplanmodifier.RequiresReplace()}`)
   - `Delete`: call `DeleteProject`
   - `ImportState`: `resource.ImportStatePassthroughID` using `name`
3. Implement `internal/provider/project_data_source.go`:
   - Schema: `name` (required), `id`/`description`/`region` (computed)
   - `Read`: call `GetProject`; error if not found
4. Register in `provider.go`
5. Write acceptance tests (`TestAccProjectResource_basic`, `TestAccProjectResource_update`, `TestAccProjectResource_import`, `TestAccProjectDataSource`)
6. Write unit tests for `project.go` JSON parsing

**Deliverable**: `terraform apply` creates/updates/deletes Akka projects. `terraform plan` detects drift. `data.akka_project` resolves existing projects.

### Phase C: Service Resource (P1 story 2)

**Goal**: Full lifecycle for `akka_service` including async deploy polling and pause/expose toggles.

1. Implement `internal/akka/service.go`:
   - `ServiceModel` struct
   - `DeployService(ctx, name, project, image, region string, replicas int, env map[string]string) (*ServiceModel, error)` — invokes `akka services deploy`
   - `GetService(ctx, name, project string) (*ServiceModel, error)`
   - `DeleteService(ctx, name, project string) error`
   - `PauseService(ctx, name, project string) error` / `ResumeService(...) error`
   - `ExposeService(ctx, name, project string) error` / `UnexposeService(...) error`
   - `WaitForReady(ctx context.Context, name, project string, interval, timeout time.Duration) error` — polls `GetService` until `status == "Ready"` or context deadline exceeded
2. Implement `internal/provider/service_resource.go`:
   - Schema per `data-model.md`; `paused` and `exposed` as bool attributes with `planmodifier` to detect changes; `status` and `hostname` computed
   - `timeouts` block using `terraform-plugin-framework-timeouts`
   - `Create`: `DeployService` → `WaitForReady` → populate computed attrs
   - `Read`: `GetService`; `NotFoundError` → remove from state
   - `Update`: handle image/env/replicas (re-deploy + wait), paused toggle, exposed toggle independently
   - `Delete`: `DeleteService`
   - `ImportState`: composite ID `<project>/<name>`
3. Implement `internal/provider/service_data_source.go`
4. Register in `provider.go`
5. Write acceptance tests and unit tests for `WaitForReady` (using a mock that transitions through states)

**Deliverable**: `terraform apply` deploys services; pause/resume and expose/unexpose work in-place; polling correctly handles `UpdateInProgress` state.

### Phase D: Secret Resource (P2 story 3)

**Goal**: All 5 secret types with sensitive field handling.

1. Implement `internal/akka/secret.go`:
   - `SecretModel` struct (name, type, project — no value fields returned by platform)
   - `CreateSecret(ctx, SecretCreateRequest) error` — type-switch on `Type` to build correct CLI flags
   - `GetSecret(ctx, name, project string) (*SecretModel, error)` — existence check only
   - `DeleteSecret(ctx, name, project string) error`
   - `SecretCreateRequest` struct with all type-variant fields
2. Implement `internal/provider/secret_resource.go`:
   - Schema per `data-model.md`: all value fields `Sensitive: true`
   - Type-specific validation in `ConfigValidators()`: enforce required fields per type
   - `Create`: `CreateSecret`
   - `Read`: `GetSecret`; not-found → remove from state; value fields are not refreshed (write-only semantic — stored value in state is authoritative)
   - `Update`: forces recreation for all changes (secrets are immutable after creation; `planmodifier.RequiresReplace()` on all value fields and `type`)
   - `Delete`: `DeleteSecret`
   - `ImportState`: composite ID `<project>/<name>`; post-import, value fields will be blank — practitioner must add `lifecycle { ignore_changes = [value] }` or re-apply
3. Register in `provider.go`
4. Write acceptance tests for each secret type; confirm sensitive values are not in plan output

**Deliverable**: All 5 secret types create/delete correctly; `terraform plan` redacts all sensitive fields.

## Key Risks and Mitigations

| Risk | Mitigation |
|---|---|
| Akka CLI JSON output schema is undocumented | Determine field names empirically in Phase A/B by running `akka projects get -o json` against a test org. Centralise all JSON struct definitions in `internal/akka/`. |
| Secret values not returned by `akka secrets get` | Treat secrets as write-only after creation. Store configured value in state; skip refresh for value fields. Document this behaviour in the resource docs. |
| `akka services deploy` has no `--wait` flag | Implemented in Phase C via `WaitForReady` polling. Configurable timeout prevents infinite waits. |
| No `AKKA_ORGANIZATION` env var | Pass `--organization` explicitly on every CLI invocation. Document this difference from other env var patterns. |
| CLI version changes break JSON output | Add a CLI version check in `Configure()` (warn if version is below a tested minimum). |
