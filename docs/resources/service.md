# akka_service

Deploys and manages an Akka service. A service is a container workload running inside an Akka
project. The provider waits for the service to reach `Ready` status before marking the resource
as created.

## Timeouts

The `timeouts` block controls how long the provider waits during service lifecycle operations:

| Operation | Default  |
|-----------|----------|
| `create`  | 20m      |
| `update`  | 20m      |
| `delete`  | 10m      |

## Example Usage

```terraform
resource "akka_service" "example" {
  name    = "my-service"
  project = akka_project.example.name
  image   = "docker.io/myorg/app:1.0"

  replicas = 2
  exposed  = true

  env = {
    APP_ENV   = "production"
    LOG_LEVEL = "info"
  }

  timeouts {
    create = "15m"
    update = "15m"
  }
}

output "service_hostname" {
  value = akka_service.example.hostname
}
```
