# akka_secret

Creates and manages an Akka secret. Secrets store credentials, certificates, and keys used by
services at runtime.

## Supported Secret Types

| Type         | Required fields                  | Description                          |
|--------------|----------------------------------|--------------------------------------|
| `symmetric`  | `value`                          | Symmetric encryption key or password |
| `asymmetric` | `private_key`, `public_key`      | RSA / EC key pair                    |
| `generic`    | `value`                          | Arbitrary opaque byte string         |
| `tls`        | `certificate`, `private_key`     | TLS certificate + private key        |
| `tls-ca`     | `ca_certificate`                 | CA certificate bundle                |

> **Note**: Secret values are write-only. The Akka platform does not return secret values after
> creation, so Terraform cannot detect value drift. Changing a value field forces replacement of
> the resource.

## Example Usage

```terraform
# Symmetric secret (JWT signing key)
resource "akka_secret" "jwt" {
  name    = "jwt-signing-key"
  project = akka_project.example.name
  type    = "symmetric"
  value   = var.jwt_signing_key
}

# TLS secret
resource "akka_secret" "tls" {
  name        = "api-tls"
  project     = akka_project.example.name
  type        = "tls"
  certificate = file("certs/api.pem")
  private_key = var.tls_private_key
}

# Generic secret (database password)
resource "akka_secret" "db" {
  name    = "db-password"
  project = akka_project.example.name
  type    = "generic"
  value   = var.db_password
}

variable "jwt_signing_key" { sensitive = true }
variable "tls_private_key" { sensitive = true }
variable "db_password" { sensitive = true }
```

## Schema

### Required

- `name` (String)
- `type` (String) Secret type: symmetric, asymmetric, generic, tls, tls-ca.

### Optional

- `ca_certificate` (String) CA certificate for tls-ca type (PEM).
- `certificate` (String) Certificate for tls type (PEM).
- `external_provider` (String) External provider: aws, azure, gcp.
- `external_reference` (String) External secret reference (ARN, URI, or name).
- `organization` (String)
- `private_key` (String, Sensitive) Private key for asymmetric and tls types (PEM).
- `project` (String)
- `public_key` (String) Public key for asymmetric type (PEM).
- `value` (String, Sensitive) Secret value for symmetric and generic types.

### Read-Only

- `id` (String) The ID of this resource.

## Import

Import an existing secret using `<project>/<name>`:

```shell
terraform import akka_secret.jwt my-project/jwt-signing-key
```

After import, populate the value fields in your configuration — Terraform cannot read them from
the platform.
